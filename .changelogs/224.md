### Added
- Bounded CI failure remediation loop for `merge-prs` Ship phase — classifies failures into 8 categories (lint-format, type-error, test-flaky, test-real, build, dependency-install, timeout-infra, unknown) and applies category-specific fixes up to 3 attempts within a 10-minute wall-clock budget (GH#224)
- New shared reference `skills/merge-prs/references/best-effort-remediation.md` — bounded-attempt loop skeleton, mode-aware escalation contract, announcement templates, skip/pause/escalate decision table, wall-clock tracking guidance. Siblings #225 and #226 will import this unchanged.
- New reference `skills/merge-prs/references/ci-remediation.md` — CI-specific category detection heuristics, fix strategies, flake handling policy, `fix(ci):` commit message contract, and `gh pr checks` polling behavior
- New `merge.ci_remediation` config section in `.feature-flow.yml` — `max_attempts` (default 3), `max_wall_clock_minutes` (default 10), `ci_poll_interval_seconds` (default 30, minimum 10)
- `ci_remediation.*` fields added to `merge-prs` Config table for discoverability

### Changed
- `merge-prs` SKILL.md Step 4a CI handling replaced: one-shot "investigate once, fix trivial, skip unfixable" is now a bounded diagnosis-and-fix loop. Skip is the last resort, not the first response.
- Error recovery table collapsed from two `CI failing, trivial fix` / `CI failing, unfixable` rows into one `CI failing` row that references the bounded loop
- Flakes are first-class: a transient timeout/network failure gets a single automatic re-run via `gh run rerun --failed` before being classified as a real test failure
- Commits made by the remediation loop use a `fix(ci):` prefix so they are greppable in `git log` and squash cleanly

### Notes
- Markdown + YAML-comment-only change — no executable code, no new tests, no runtime dependencies
- This is the first of three sibling merge-prs improvements (#224 CI remediation, #225 merge conflict ladder, #226 PR review triage). The shared `best-effort-remediation.md` reference file is created here and will be reused by #225 and #226 without modification.
