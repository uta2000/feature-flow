### Added
- **Reference-integrity CI check for skill files (#255)** — New `scripts/check-skill-references.sh` (bash) scans every `*.md` file under `skills/` for `Read \`references/...\`` and `Read \`../../references/...\`` instructions, resolves each path relative to the nearest ancestor `SKILL.md` directory (the skill-root), and fails with a per-line error message if any referenced file is missing. Multiple references on a single line are supported (`grep -noE` emits one match per occurrence) — important because `skills/merge-prs/SKILL.md:105` already carries two references on one line. The script anchors on the literal `Read \`` token so plain prose mentioning a path or non-reference `Read \`...\`` instructions (e.g., `Read \`.feature-flow.yml\``) are not flagged. New GitHub Actions workflow at `.github/workflows/check-references.yml` runs the script on every `pull_request` and on `push: branches: [main]` using `actions/checkout@v6` and `ubuntu-latest`. This is the first GitHub Actions workflow in the repo. Style mirrors the only other shell script in the codebase (`tests/start/quick_path/check-drift.sh`): `set -u`, no `set -e`, `git rev-parse --show-toplevel` anchor, error-counter accumulator, `✓`/`✗` summary line. Out of scope for this PR (filed as follow-ups if needed): wiring `check-drift.sh` into the same workflow, scanning for unreferenced files in `references/` directories, fence-aware matching (no false-positive cases exist in the current codebase, so it would be speculative complexity). Empirical validation pre-merge: clean repo exits 0; injected broken `references/X.md` exits 1; injected broken `../../references/X.md` resolves via skill-root walk and exits 1; prose mention without `Read \`` prefix exits 0; injected two broken refs on one line produces two error lines.
