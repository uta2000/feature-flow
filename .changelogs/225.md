### Added
- `merge-prs` conflict resolution now follows a 4-tier ladder: Tier 1 (trivial auto-resolve) → Tier 2 (attempt-with-test-verification, NEW) → Tier 3 (diff-presentation, renamed from "Behavioral Conflicts") → Tier 4 (skip). Tier 2 attempts an additive union merge for structurally-independent both-sided modifications and verifies safety by running the project test suite before committing. Tier 3 always pauses regardless of mode (safety invariant preserved).
- `merge.conflict_resolution.test_command` config field — optional override for the Tier 2 test runner. If unset, stack-based detection is used (pnpm/yarn/npm for node-js; pytest for python).
- `merge.conflict_resolution.test_timeout_minutes` config field — hard wall-clock timeout for Tier 2 test verification (default: 5 minutes, minimum: 1).
- Structural Independence Gate: determines whether a both-sided modification with behavioral keywords is eligible for Tier 2 (non-overlapping changes → gate passes) or must escalate directly to Tier 3 (semantic overlap → gate fails).
- Test Runner Discovery: reusable helper in `conflict-resolution.md` that detects the project's test command via explicit config or stack-based lockfile/config-file heuristics.
- Timeout Detection: detects GNU `timeout`, falls back to `gtimeout` (Homebrew macOS), then to a bash background-job + kill pattern as a last resort.
- Example 7 in `conflict-resolution.md`: demonstrates a Tier 2 success path and a Tier 2-to-Tier 3 escalation path with realistic conflict markers.

### Changed
- Renamed "Behavioral Conflicts" section to "Tier 3: Diff Presentation (Always Pauses)" in `conflict-resolution.md`. All semantics preserved; "Accept proposed" added as the first `AskUserQuestion` option when Tier 2 produced a resolution.
- `SKILL.md` §Conflict Resolution summary now describes the 4-tier ladder explicitly. Worktree flow at lines 223-249 and the CWD Safety Guard block are preserved byte-for-byte.
- `SKILL.md` Error Recovery table: the "Merge conflict, behavioral" row is replaced with two rows — "Merge conflict, structurally independent → Tier 2" and "Merge conflict, semantic overlap → Tier 3 (pause)".
- Structure Classification routing now sends `both-sided modification` through the structural independence gate: gate passes → Tier 2, gate fails → Tier 3. `unknown/malformed` conservatively routes to Tier 3.
